# Content Briefs

This directory holds content briefs — structured JSON files that feed `src/scripts/content-draft.ts` to produce draft MDX blog posts.

## Lifecycle

```
src/scripts/keyword-seeds.json
         │
         ▼  (weekly cron: .github/workflows/content-pipeline.yml)
  keyword-brief-generator.ts
         │  creates JSON in...
         ▼
  briefs/queue/<slug>.json  ← "queued"  (auto-generated, not committed)
         │
         ▼  (same workflow, chained step)
  content-draft.ts
         │  writes MDX to...
         ▼
  src/content/blog/<slug>.mdx   (status: draft)
         │
         ▼  (auto-opens a draft PR labeled `needs-human-review`)
  PR review → redline → flip `status: draft` → `status: published` → merge
         │
         ▼
  Live on /blog
```

## Directory layout

```
briefs/
├── README.md                     (this file)
├── sample-class-4-shingle.json   (checked-in example — use for test runs)
└── queue/                        (auto-generated by the brief generator; not committed)
    └── <slug>.json
```

## Running the pipeline manually

### Option 1 — generate a brief + draft in one shot
```bash
BRIEF_PATH=$(pnpm tsx src/scripts/keyword-brief-generator.ts)
pnpm tsx src/scripts/content-draft.ts --brief="$BRIEF_PATH"
```

### Option 2 — draft from an existing brief
```bash
pnpm tsx src/scripts/content-draft.ts --brief=briefs/sample-class-4-shingle.json
```

### Option 3 — write a one-off brief by hand
1. Copy `sample-class-4-shingle.json` to a new file in this directory.
2. Edit the fields (see schema below).
3. Run: `pnpm tsx src/scripts/content-draft.ts --brief=briefs/your-new-brief.json`.

## Brief schema

```ts
{
  "title": string,
  "slug": string,
  "cluster":
    "inspection" | "replacement" | "repair" | "storm-damage" |
    "insurance-claims" | "materials" | "maintenance" | "buyer-seller" |
    "local-weather" | "hoa" | "inspector-peers",
  "persona": ["homeowner" | "agent" | "insurance" | "storm" | "hoa" | "inspector", ...],
  "relatedCity": "the-woodlands" | "spring" | "tomball" | "magnolia" | "conroe" | "cypress",
  "targetKeyword": string,
  "secondaryKeywords": string[],
  "angle": string,
  "mustInclude": string[],
  "mustNotInclude": string[],
  "wordCountTarget": number
}
```

## Before running

Make sure your env has `ANTHROPIC_API_KEY` set. The drafter uses Claude Sonnet by default — about $0.03-$0.08 per brief expansion + $0.15-$0.25 per full-post draft.

## Quality gates

Every draft MDX writes with `status: "draft"`, so it won't ship to production until a human flips it to `published`. Enforcement at:
1. `src/content.config.ts` — schema has `status` defaulting to `"draft"`.
2. `src/pages/blog/index.astro` — filters out non-published.
3. `src/pages/blog/[...slug].astro` — `getStaticPaths` only includes `status === "published"`.
